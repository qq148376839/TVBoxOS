## 问题
默认播放器（System MediaPlayer, type=0）播放某些内容时黑屏。切换到 ExoPlayer/IJK 后正常。
ExoPlayer 版本为 2.14.2（2021 年），已停止维护 5 年，需要升级到 Media3 1.10.0。

## 根因分析
1. System MediaPlayer 对 H.265/HEVC、特殊容器（MKV、FLV）、部分 HLS 流的支持因设备/Android 版本而异
2. ExoPlayer 2.14.2 虽然能播，但缺少 5 年间的 bug 修复、AV1/HDR 支持、更好的 HLS/DASH 处理
3. 默认播放器设为能力最弱的 System MediaPlayer 是不合理的默认值

## 方案

### 第一步：依赖迁移

`player/build.gradle` 替换依赖：

```
删除:
  com.google.android.exoplayer:exoplayer-core:2.14.2
  com.google.android.exoplayer:exoplayer-ui:2.14.2
  com.google.android.exoplayer:exoplayer-dash:2.14.2
  com.google.android.exoplayer:exoplayer-hls:2.14.2
  com.google.android.exoplayer:exoplayer-rtsp:2.14.2
  com.google.android.exoplayer:extension-rtmp:2.14.2
  com.google.android.exoplayer:extension-okhttp:2.14.2

添加:
  androidx.media3:media3-exoplayer:1.10.0
  androidx.media3:media3-exoplayer-dash:1.10.0
  androidx.media3:media3-exoplayer-hls:1.10.0
  androidx.media3:media3-exoplayer-rtsp:1.10.0
  androidx.media3:media3-datasource-okhttp:1.10.0
  androidx.media3:media3-ui:1.10.0        (如需)
  androidx.media3:media3-common:1.10.0
```

RTMP 支持：检查 Media3 是否内置 RTMP 或需要额外扩展。

### 第二步：Player 模块 ExoPlayer 适配层

**`player/src/.../exo/ExoMediaPlayer.java`**：
- `com.google.android.exoplayer2.*` → `androidx.media3.*`
- `SimpleExoPlayer` → `ExoPlayer`（通过 `new ExoPlayer.Builder(context).build()`）
- `ExoPlaybackException` → `PlaybackException`
- `Player.Listener` 包路径变更
- `VideoSize.unappliedRotationDegrees` → 检查新 API 对应方式
- `AnalyticsCollector(Clock.DEFAULT)` → 检查 Builder 是否仍需要显式传入
- 添加 `@OptIn(UnstableApi.class)` 注解

**`player/src/.../exo/ExoMediaSourceHelper.java`**：
- `ExoDatabaseProvider` → `StandaloneDatabaseProvider`
- `DefaultDataSourceFactory` → `DefaultDataSource.Factory`
- `OkHttpDataSource.Factory` 包路径变更
- `RtmpDataSourceFactory` → 检查 Media3 对应
- `MediaItem.fromUri()` 保持不变（Media3 通用）
- `Util.getUserAgent()` → 可能已移除，需替换
- 所有 `com.google.android.exoplayer2.source.*` → `androidx.media3.exoplayer.source.*`
- 所有 `com.google.android.exoplayer2.upstream.*` → `androidx.media3.datasource.*`

**`player/src/.../exo/ExoMediaPlayerFactory.java`**：
- 包路径更新

### 第三步：App 模块适配

**`app/src/.../player/ExoPlayer.java`**（音轨选择 — 改动最大）：
- Track selection API 重大变更：
  - `DefaultTrackSelector.SelectionOverride` → `TrackSelectionOverride`
  - `params.getSelectionOverride(rendererIndex, groups)` → 新 API
  - `builder.clearSelectionOverrides(rendererIndex)` → `builder.clearOverridesOfType(trackType)`
  - `builder.setSelectionOverride(rendererIndex, groups, override)` → `builder.addOverride(new TrackSelectionOverride(...))`
  - `MappingTrackSelector.MappedTrackInfo.getTrackGroups(rendererIndex)` → 确认是否有变化
  - `TrackGroupArray` → 可能变为 `TrackGroup` 的集合
  - `Format` 字段访问方式可能变化

**`app/src/.../player/ExoMediaPlayerFactory.java`**：
- 包路径更新

**`app/src/.../util/PlayerHelper.java`**：
- 默认 `PLAY_TYPE` 从 0 (System) 改为 2 (ExoPlayer)
- 确保首次安装用户直接使用 ExoPlayer

### 第四步：DKPlayer UI 兼容性验证

`dkplayer-ui:3.3.5` 通过 `AbstractPlayer` 接口交互，不直接引用 ExoPlayer 类。
但需验证：
- `VideoView` 基类是否有任何 ExoPlayer 类型引用
- 编译后是否有未解析的类

## 改动范围
- `player/build.gradle`：依赖替换
- `player/src/.../exo/ExoMediaPlayer.java`：核心播放器适配
- `player/src/.../exo/ExoMediaSourceHelper.java`：数据源/媒体源适配
- `player/src/.../exo/ExoMediaPlayerFactory.java`：工厂包路径
- `app/src/.../player/ExoPlayer.java`：音轨选择 API 重写
- `app/src/.../player/ExoMediaPlayerFactory.java`：工厂包路径
- `app/src/.../util/PlayerHelper.java`：默认播放器改为 ExoPlayer

## 风险点
- Track selection API 变化最大，音轨切换/记忆功能需要仔细适配
- RTMP 支持在 Media3 中可能需要额外处理（ExoPlayer 2.x 的 extension-rtmp 对应是什么）
- `@OptIn(UnstableApi.class)` 注解需要在多处添加，否则 lint 报错
- DKPlayer UI 库版本是否需要同步更新

## 验证方式
- `./gradlew :player:assembleDebug` 编译通过
- `./gradlew :app:assembleNormalDebug` 编译通过
- 测试播放器切换：System/IJK/ExoPlayer 三种都正常
- 测试格式：HLS (.m3u8)、RTMP、普通 MP4、DASH
- 测试音轨切换和记忆功能
- 测试黑屏场景是否解决

## 讨论记录
- 用户确认只有 System MediaPlayer 黑屏，ExoPlayer/IJK 正常
- 讨论了方案 A（仅改默认值）和方案 B（升级 Media3），用户选择直接上 B
- Media3 最新稳定版 1.10.0（2026.03 发布）
